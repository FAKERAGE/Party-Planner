## Challenge 3: V1 Diagnosis Sheet & Keep/Change/Delete List

**Note**  
Goal: Complete a formal V1 Diagnosis Sheet that documents what your app does well, what it does poorly, and what is missing. Then identify a target user and create a Keep/Change/Delete list.

This is the most important challenge of the day. Everything you build in Lessons 2-4 will be based on what you write here.

**Steps:**

Complete the V1 Diagnosis Sheet -- copy this into a document and fill it in honestly:

| Category | Rating (1-5) | Specific Notes |
|----------|--------------|---------------|
| Event creation flow | 4 | Simple form with name, date, location, description; saves to memory. |
| Theme selection & visuals | 3 | Dropdown changes background color, but lacks thematic elements like images or icons. |
| Guest list management | 4 | Add guests with RSVP status, remove them; functional but basic. |
| Activity planner | 3 | Drag-drop from list to planned area works, but no time slots or reordering. |
| Overall layout & design | 3 | Clean and functional, but plain and not engaging for teens. |
| Would someone actually use this? | 3 | For quick basic planning, but missing key features like persistence and sharing. |

Write 3 strengths -- things that actually work well in the AI draft

1. Quick and straightforward event creation with essential fields.
2. Functional guest list with RSVP tracking and easy removal.
3. Intuitive drag-and-drop interface for adding activities.

Write 3 weaknesses -- things that feel broken, confusing, or generic

1. No data persistence - all data is lost on page refresh.
2. Theme selection is minimal, only changing background color without immersive visuals.
3. No support for organizing multiple events (e.g., folders).

Write 3 missing features -- things a real user would expect that are not there

1. Budget tracking for event costs.
2. Shareable event links or invites for guests.
3. Printable or exportable event summary.

Define your target user -- answer these questions:

Who specifically would use this app? (Not "everyone" -- pick a real type of person)
A 13-year-old grade 8 student.

When would they use it? (What situation triggers the need?)
A week before a trip or during a busy week when planning is needed quickly.

What do they care about most when planning an event?
Everything being accessible and organized for efficient planning.

What frustrates them about planning events without a tool like this?
Not knowing what activities to plan and forgetting details like guest lists.

Write your Target User Statement: "We are redesigning this for 13-year-old students who want efficient organization during busy planning weeks"

Write your Redesign Direction Statement: "Our AI version works, but it is too basic. We want to redesign it so that it feels more engaging for young teens"

Create your Keep/Change/Delete (K/C/D) list:

## MVP Scoping

MVP stands for Minimum Viable Product. It is the smallest version of your product that delivers real value to your target user. Not the smallest version you can build -- the smallest version that matters to someone.

The core interaction loop for an Event & Party Planner is:

**Create event → Add details → Manage guests → Plan activities → Preview/share**

Your MVP must complete this loop. Everything that supports the loop is must-have. Everything else is nice-to-have.
