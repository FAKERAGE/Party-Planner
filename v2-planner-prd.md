# Event & Party Planner -- V2 MVP PRD

## Target User
This app is for kids and teens who plan upcoming parties, hangouts, or gatherings. They need activities planning most, with sessions lasting 10-15 minutes. They care about activities planning and dislike forgetting who is coming. They would stop using it if the UI is hard to use.

## What We Learned from V1
What worked: Theme system.
What did not: Drag part for activities (top ones not draggable).
K/C/D: Keep - Adding activities. Change - Drag functionality. Delete - Preset activities.

## Top 3 User Needs
- [x] Need 1: Easy to navigate website
- [x] Need 2: Easy organization
- [x] Need 3: Simple guest management

## Product Positioning
A planning tool for kids that gives them organized and easy planning for upcoming events

## Must-Have Features (MVP)
- [x] Event creation form focused on themes
- [x] Guest list management with RSVP
- [x] Activity planning with time slots
- [x] Theme system that uses icons
- [x] Preview/summary view that shows everything that's gonna happen

## Nice-to-Have (only if time allows)
- [ ] Budget tracking

## What To Avoid
- Do NOT add features from the Delete list (preset activities)
- Do NOT add user accounts, login, databases, or backend
- Keep the code in a single HTML file with embedded CSS and JS
- Every feature must connect to a user need -- no decorations without purpose

## Design Direction
The visual feel should be simple to match the target user's age/context, keeping it easy to navigate and not overwhelming.
